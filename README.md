# Техническая документация проекта EduPath

Данный документ содержит подробное описание архитектуры, структуры и функциональности файлов образовательной платформы EduPath. Проект реализован с использованием стандартных веб-технологий: HTML5, CSS3 и Vanilla JavaScript.

## 1. HTML Структура (index.html)

Файл разметки отвечает за семантическое построение страницы и размещение контента.

### 1.1. Базовая структура
*   `<!DOCTYPE html>`: Объявляет тип документа как HTML5.
*   `<html lang="ru">`: Корневой элемент, атрибут `lang` указывает язык содержимого (русский).
*   `<head>`: Содержит метаданные, не отображаемые напрямую на странице.
    *   `<meta charset="UTF-8">`: Кодировка символов.
    *   `<meta name="viewport">`: Настройка отображения на мобильных устройствах (масштабирование).
    *   `<title>`: Заголовок вкладки браузера.
    *   `<link>`: Подключение внешних ресурсов (стили, шрифты).
*   `<body>`: Видимая часть документа.

### 1.2. Header (Шапка сайта)
*   `<header class="header">`: Семантический тег для вводной части сайта.
    *   `.header-content`: Контейнер для выравнивания элементов шапки.
    *   `.logo`: Блок логотипа.
        *   `<h1>`: Название проекта.
        *   `.logo-tag`: Дополнительный текстовый дескриптор.
    *   `.development-notice`: Уведомление о статусе разработки.
    *   `.donate-btn`: Кнопка вызова модального окна поддержки.

### 1.3. Main Content (Основной контент)
*   `<main class="main-content">`: Контейнер для основного содержимого страницы.
*   `<section class="content-section">`: Логические разделы страницы.
    *   `.section-title`: Заголовки разделов.
    *   `.title-icon`: Иконки рядом с заголовками.

#### Секция Flashcards (Флеш-карточки)
*   `.flashcards-grid`: Сетка для размещения карточек.
*   `.flashcard`: Контейнер отдельной карточки.
    *   `.flashcard-inner`: Внутренняя обертка для 3D-трансформации.
    *   `.flashcard-front`: Лицевая сторона (вопрос/слово).
    *   `.flashcard-back`: Обратная сторона (ответ/перевод).
    *   `.card-label`: Метка категории на карточке.
    *   `.card-text`: Основной текст карточки.
    *   `.card-flip-hint`: Подсказка о возможности переворота.

#### Секция Grammar Rules (Правила грамматики)
*   `.topic-layout`: Сетка для разделения теории и практики.
*   `.content-block`: Универсальный блок контента.
    *   `.block-icon`: Иконка блока.
    *   `.rule-list`, `.example-list`: Списки правил и примеров.
*   `.grammar-table`: Таблица для отображения грамматических форм.
    *   `<thead>`, `<tbody>`, `<tr>`, `<th>`, `<td>`: Стандартные элементы таблицы.
*   `.inline-flashcard-right`: Встроенная мини-карточка внутри текста.
*   `.mini-flashcard-block`: Блок сетки мини-карточек.

#### Секция Exercises (Упражнения)
*   `.examples-block`: Контейнер для упражнений.
*   `.exercise-item`: Отдельное задание.
    *   `.exercise-question`: Текст вопроса.
    *   `.input-exercise`: Блок ввода текста.
        *   `.answer-input`: Поле ввода ответа пользователем.
        *   `.check-btn`: Кнопка проверки ответа.
        *   `.show-answer-btn`: Кнопка показа правильного ответа.
        *   `.correct-answer`: Блок отображения верного ответа.
    *   `.multiple-choice-options`: Блок вариантов выбора.
        *   `.choice-option`: Отдельный вариант ответа.
        *   `.option-marker`: Маркер варианта (A, B, C).
        *   `.validate-btn`: Кнопка валидации выбора.
        *   `.explanation-text`: Блок пояснения после проверки.

#### Секция Materials (Материалы)
*   `.materials-grid`: Сетка ссылок на дополнительные ресурсы.
*   `.content-block`: Карточки материалов с описанием.
    *   `.block-footer`: Подвал карточки с тегами.
    *   `.block-tag`: Категория материала.
    *   `.block-count`: Количество элементов.

### 1.4. Footer (Подвал)
*   `<footer class="footer">`: Семантический тег для нижней части сайта.
*   `.footer-content`: Сетка колонок подвала.
*   `.footer-section`: Отдельная колонка (Авторы, Проект, Лицензия).
*   `.author-link`: Ссылки на контакты авторов.

### 1.5. Modal (Модальное окно)
*   `.modal`: Контейнер модального окна (скрыт по умолчанию).
    *   `.modal-backdrop`: Затемнение фона.
    *   `.modal-content`: Основное окно сообщения.
    *   `.modal-close`: Кнопка закрытия.
    *   `.donate-options`: Сетка вариантов сумм.
    *   `.donate-option`: Кнопка выбора суммы.
    *   `.donate-badge`: Метка "Популярное".

## 2. CSS Стилизация (styles.css)

Файл стилей отвечает за визуальное представление, адаптивность и анимации.

### 2.1. Переменные (CSS Variables)
Объявлены в селекторе `:root` для глобального управления темой.
*   `--bg-primary`, `--bg-secondary`, `--bg-card`: Цвета фона различных уровней.
*   `--accent-primary`, `--accent-gradient`: Основные акцентные цвета и градиенты.
*   `--text-primary`, `--text-secondary`: Цвета текста для заголовков и основного контента.
*   `--success`, `--error`, `--warning`: Цвета состояний (верно, ошибка, предупреждение).
*   `--shadow-*`: Наборы теней для глубины интерфейса.
*   `--radius-*`: Радиусы скругления углов.
*   `--transition-*`: Параметры временных функций для анимаций.

### 2.2. Базовые стили и Сброс
*   `* { margin: 0; padding: 0; box-sizing: border-box; }`: Глобальный сброс отступов и включение удобной модели коробок.
*   `body`: Установка шрифта, фона, цвета текста и запрет горизонтального скролла (`overflow-x: hidden`).
*   `body::before`: Декоративный фоновый градиент с фиксированным позиционированием.

### 2.3. Layout и Сетки
*   `.header`: `position: sticky` фиксирует шапку при прокрутке. `backdrop-filter: blur` создает эффект матового стекла.
*   `.flashcards-grid`, `.materials-grid`: Использование `display: grid` с `repeat(auto-fit, minmax(...))` для автоматической адаптивности колонок.
*   `.topic-layout`: Двухколоночная сетка для разделения теории и практики.
*   `.footer-content`: Сетка из трех колонок для подвала.

### 2.4. Компоненты и Эффекты
*   `.content-block`:
    *   `::before`: Псевдоэлемент для создания цветной полосы сверху при наведении (`transform: scaleX`).
    *   `:hover`: Изменение тени и смещение блока вверх.
*   `.flashcard` (3D Переворот):
    *   `perspective: 1000px`: Создает глубину для 3D-эффекта на родителе.
    *   `.flashcard-inner`: `transform-style: preserve-3d` сохраняет 3D-пространство для детей.
    *   `.flipped .flashcard-inner`: `transform: rotateY(180deg)` поворачивает карточку.
    *   `.flashcard-front`, `.flashcard-back`: `backface-visibility: hidden` скрывает обратную сторону элемента при повороте.
*   `.grammar-table`: Стилизация границ, отступов и цветов строк для читаемости данных.
*   `.answer-input`:
    *   `.correct`: Зеленая граница и фон при верном ответе.
    *   `.incorrect`: Красная граница и анимация тряски при ошибке.
*   `.modal`:
    *   `position: fixed`: Поверх всего контента.
    *   `.active`: Класс для отображения (меняет `display: none` на `flex`).

### 2.5. Анимации (@keyframes)
*   `shake`: Покачивание элемента влево-вправо (используется при ошибке).
*   `slideIn`: Плавное появление элемента снизу вверх (используется для ответов).
*   `modalSlideIn`: Появление модального окна с масштабированием.
*   `ripple`: Эффект волны при клике на кнопку доната (добавляется через JS).

### 2.6. Адаптивность (Media Queries)
*   `@media (max-width: 1200px)`: Перестроение сетки тем в одну колонку.
*   `@media (max-width: 900px)`: Изменение количества колонок в сетке карточек.
*   `@media (max-width: 600px)`: Мобильная версия.
    *   Уменьшение шрифтов и отступов.
    *   Изменение сетки мини-карточек.
    *   Адаптация таблиц (горизонтальный скролл или блочное отображение).
    *   Перестроение кнопок и полей ввода в упражнениях.

## 3. JavaScript Логика (script.js)

Файл скриптов обеспечивает интерактивность, валидацию форм и управление состоянием интерфейса.

### 3.1. Управление Карточками
*   `function flipCard(card)`:
    *   **Назначение:** Переключает класс `flipped` у переданного элемента.
    *   **Параметры:** `card` (DOM-элемент карточки).
    *   **Механизм:** `card.classList.toggle('flipped')`. CSS обрабатывает визуальный переворот.

### 3.2. Валидация Текстовых Упражнений
*   `function checkTextAnswer(button, correctAnswers)`:
    *   **Назначение:** Проверяет ввод пользователя against массив правильных ответов.
    *   **Параметры:** `button` (элемент кнопки проверки), `correctAnswers` (массив строк).
    *   **Логика:**
        1.  Находит родительский элемент `.exercise-item`.
        2.  Получает значение из `.answer-input`, приводит к нижнему регистру.
        3.  Сравнивает с массивом правильных ответов.
        4.  При успехе: добавляет класс `.correct`, скрывает кнопку подсказки.
        5.  При ошибке: добавляет класс `.incorrect`, показывает кнопку подсказки.
*   `function showCorrectAnswer(button)`:
    *   **Назначение:** Переключает видимость блока с правильным ответом.
    *   **Логика:** Находит `.correct-answer` внутри задания и переключает класс `.visible`.

### 3.3. Упражнения с Выбором Варианта
*   `function selectOption(option)`:
    *   **Назначение:** Выделяет выбранный вариант ответа.
    *   **Логика:**
        1.  Находит контейнер вариантов.
        2.  Сбрасывает классы `selected`, `correct`, `incorrect` у всех вариантов.
        3.  Добавляет `selected` текущему элементу.
        4.  Скрывает текст пояснения при новом выборе.
*   `function validateMultipleChoice(button)`:
    *   **Назначение:** Проверяет выбранный вариант.
    *   **Логика:**
        1.  Находит выбранный элемент `.choice-option.selected`.
        2.  Если ничего не выбрано, запускает анимацию тряски кнопки.
        3.  Проверяет атрибут `data-correct`.
        4.  Добавляет классы `.correct` или `.incorrect`.
        5.  Отображает текст пояснения из атрибута `data-explanation`.

### 3.4. Модальное окно Пожертвований
*   `function showDonateModal()`:
    *   **Назначение:** Открывает модальное окно.
    *   **Логика:** Добавляет класс `.active` окну и блокирует скролл страницы (`body.style.overflow = 'hidden'`).
*   `function closeDonateModal()`:
    *   **Назначение:** Закрывает модальное окно.
    *   **Логика:** Убирает класс `.active` и возвращает скролл.
*   `function selectDonate(amount)`:
    *   **Назначение:** Обрабатывает выбор суммы.
    *   **Логика:**
        1.  Вычисляет координаты клика для эффекта волны (ripple).
        2.  Создает временный элемент `span` с анимацией.
        3.  Удаляет элемент после анимации.
        4.  Показывает уведомление `alert`.
        5.  Закрывает модальное окно.
*   `document.addEventListener('keydown', ...)`:
    *   **Назначение:** Закрытие модального окна по клавише Escape.

### 3.5. Навигация и Наблюдение
*   `document.querySelectorAll('a[href^="#"]')`:
    *   **Назначение:** Плавная прокрутка к якорным ссылкам.
    *   **Метод:** `target.scrollIntoView({ behavior: 'smooth' })`.
*   `IntersectionObserver`:
    *   **Назначение:** Анимация появления элементов при скролле.
    *   **Настройки:** `threshold: 0.1` (срабатывает при 10% видимости).
    *   **Логика:**
        1.  Изначально скрывает элементы (`opacity: 0`, `translateY(20px)`).
        2.  При попадании в область видимости возвращает стили (`opacity: 1`, `translateY(0)`).
    *   **Целевые элементы:** `.content-block`, `.examples-block`, `.flashcard` и другие.

## 4. Взаимодействие Компонентов

### 4.1. HTML и CSS
HTML предоставляет классы-хуки (например, `.flipped`, `.correct`), которые CSS использует для изменения визуального состояния.
*   **Пример:** HTML элемент `.flashcard` получает класс `.flipped` от JS. CSS правило `.flashcard.flipped .flashcard-inner` применяет трансформацию поворота.

### 4.2. HTML и JavaScript
JavaScript использует селекторы для поиска элементов по классам и ID, определенных в HTML.
*   **Пример:** `document.getElementById('donateModal')` обращается к элементу `<div id="donateModal">` в HTML.
*   **Пример:** Атрибуты `data-correct` и `data-explanation` в HTML читаются через JS метод `getAttribute()`.

### 4.3. CSS и JavaScript
JavaScript динамически изменяет стили или добавляет классы, которые запускают CSS-анимации.
*   **Пример:** JS создает элемент `span` для эффекта ripple, а CSS анимация `@keyframes ripple` управляет его расширением и исчезновением.
*   **Пример:** JS меняет `style.overflow` у `body`, чтобы заблокировать прокрутку, когда CSS модального окна активно.

## 5. Заключение

Проект представляет собой статическое одностраничное приложение (SPA), не требующее серверной обработки для базовой функциональности.
*   **Масштабируемость:** Добавление нового контента осуществляется путем копирования блоков HTML с соответствующими классами.
*   **Поддержка:** Использование CSS-переменных позволяет легко изменить цветовую схему всего проекта в одном месте.
*   **Производительность:** Отсутствие тяжелых библиотек обеспечивает быструю загрузку. Анимации реализованы через CSS Transform и Opacity для использования аппаратного ускорения.
